使用 Vite + Tailwind CSS v4 搭建个人网站

2026-06-01 · 阅读约 5 分钟

Vite 是目前最流行的前端构建工具之一，Tailwind CSS 是实用优先的 CSS 框架。 本文介绍如何将两者结合，搭建一个轻量、高性能的个人静态博客。

为什么选择 Vite + Tailwind

  • Vite 提供极快的冷启动和 HMR，开发体验流畅。
  • Tailwind CSS v4 使用 CSS-first 配置，无需 PostCSS 管道，构建速度更快。
  • 两者结合可以直接编写 HTML，适合静态博客场景。

项目结构

清晰的目录结构让项目在持续扩展中保持可维护：

huoxin_protflio/
├── index.html              # 首页
├── blog/
│   ├── index.html          # 博客列表
│   └── posts/              # 文章（HTML 文件）
│       └── hello-world.html
├── src/
│   ├── style.css           # Tailwind 导入 + 自定义主题
│   └── main.js             # 入口文件
├── vite.config.js          # Vite + Tailwind 插件 + 多页面
├── tailwind.config.js      # 编辑器兼容参考
└── package.json

配置 Tailwind CSS v4

v4 只需要在 CSS 文件中导入一行即可启用，主题定制放在 @theme 块中： 

@import 'tailwindcss';

@theme {
  --font-sans: 'Inter', system-ui, sans-serif;
  --color-accent: #2563eb;
  --color-bg: #f8fafc;
  --color-text: #1e293b;
}

然后在 vite.config.js 中注册 Tailwind 插件： 

import tailwindcss from '@tailwindcss/vite'

export default defineConfig({
  plugins: [tailwindcss()],
})
Tailwind v4 的 CSS-first 配置消除了对 postcss.config.js 的依赖，大幅简化了构建管道。

组织博客文章

对于纯静态站点，推荐以下两种组织方式：

  1. 直接写 HTML — 最简单，每篇文章一个 .html 文件，可直接使用 Tailwind 类。
  2. Markdown + 构建时转换 — 将 .md 文件放在 blog/posts/ 目录，通过 Vite 插件在构建时转为 HTML。

建议初期直接用 HTML，文章多了再引入 Markdown 转换。

总结

Vite + Tailwind CSS v4 的组合非常适合搭建个人博客：零运行时开销、极致的开发体验、产物体积小。模板搭好后，只需专注内容创作即可。
